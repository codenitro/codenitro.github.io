Production Ready Asp.Net Core WebAPI

Building a production ready Asp.Net Core WebAPI using VS Code

 Posted on February 18, 2021  |  2 minutes  |  264 words  |  Aseem Gaurav

Now that dotnet core version 5 has been available for some time, looking at its progress, adoption and community support, I truly believe that it is ready for prime time. It is one of the best open source frameworks to build all kinds of applications, including web api’s or backend services.

There are lots of resources available on building api’s with .Net core but very few cover every aspect of it. Community has produced great articles, videos and blogs on how to integrate various bits and pieces while building an api but there are very few which cover most of these in a single application.

In this series of blog posts, I will try and cover all aspects of building an api that is ready for production and at the same time offers great developer productivity by leveraging technologies like docker. Since I have been using Open Source software and frameworks for a long time, I want to achieve the above goals using free and open source tools only for development like VS Code instead of Visual Studio.

The application that we’re going to build will be a simple one as we dont want to spend time in understanding the domain of the application. Instead, be keeping the application simple, we’ll be able to concentrate more on the technical side of dotnet core. We will create a few simple API’s that will store the users profile information like his name, personal details, addresses, and saved card details.
