Asp.Net Core WebAPI - New Project

Part 2 - Creating a new asp.net core web api project

 Posted on February 18, 2021  |  3 minutes  |  530 words  |  Aseem Gaurav

After installing .Net Core Framework, you will be able to run dotnet commands from the command line. To check your installation is working fine, run the version command and it should show the version of .net core installed.

  1. Open a command prompt window
  2. Run the command dotnet --version It will show an output as shown below. The version might differ in your case.
C:\GitHub>dotnet --version
5.0.103

When using vs code, a lot of dotnet functionality will be accessed via the dotnet command line. Running the command dotnet new will show you all the availbale project types. What we are interested in the WebAPI project.

C:\GitHub>dotnet new
Getting ready...
Templates                                         Short Name               Language          Tags
--------------------------------------------      -------------------      ------------      ----------------------
...
ASP.NET Core Web API                              webapi                   [C#], F#          Web/WebAPI
...

From the command line, go tohe folder where you want to create your project. I am using the folder C:\GitHub\aspnet-core-webapi. Once you are inside the folder from the command prompt run the below command to create a web api project.

C:\GitHub\aspnet-core-webapi>dotnet new webapi -o Profile.API
The template "ASP.NET Core Web API" was created successfully.

Processing post-creation actions...
Running 'dotnet restore' on Profile.API\Profile.API.csproj...
  Determining projects to restore...
  Restored C:\GitHub\aspnet-core-webapi\Profile.API\Profile.API.csproj (in 6.25 sec).
Restore succeeded.

Lets have a look at the command that we just ran. dotnet new webapi -o Profile.API Here dotnet is calling the dotnet CLI. new webapi tells dotnet cli to create a new webapi project -o Profile.API tells dotnet cli to create all files in a folder named Profile.API

To see all available options for the new command you can run dotnet new --help. This works with all command names so you can get help for any command.

Running the command successfully will create the below folder structure:

If you are coming from Visual Studio or previous .net versions, you will notice that dotnet core does not create a .sln or the solution file. In classic .net framework, solution file is used to keep a reference to all the projects. In case of .net core, this is not needed, although you can create one.

now you can run the project either from the command line or from VS code. Lets do if from VS code for now. To run, all you need to do is press F5. If there are no errors, you should be able to access the api by going to the url:

https://localhost:5001/WeatherForecast

Dotnet core webapi comes with bilt in support for Swagger. If you go to the Startup.cs class, you can see that the necessary setup for swagger has already been done for you. Startup.cs

services.AddSwaggerGen(c =>
{
    c.SwaggerDoc("v1", new OpenApiInfo { Title = "Profile.API", Version = "v1" });
});

if (env.IsDevelopment())
{
    app.UseDeveloperExceptionPage();
    app.UseSwagger();
    app.UseSwaggerUI(c => c.SwaggerEndpoint("/swagger/v1/swagger.json", "Profile.API v1"));
}

Swagger provides a way to document your api’s and an easy way to try it out. You can access swagger by going to the url https://localhost:5001/swagger/

For more details on swagger check out: Swagger
